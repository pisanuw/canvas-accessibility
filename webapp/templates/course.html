What does this tool do?

This tool automatically finds and fixes common accessibility issues in your Canvas course — including PDF metadata, Word and PowerPoint documents, and Canvas page HTML. It uses the Canvas API with your personal token to read and update course content. If you provide an Anthropic API key, it can also generate alt text for images using AI.

Issues this tool fixes

Canvas Pages (HTML)
  • Empty headings — Removes heading tags that contain no text, which confuse screen readers.
  • Heading order — Corrects heading levels that skip (e.g. H1 → H3) so the outline is sequential.
  • Heading presence — Adds a heading to pages that have none, giving screen reader users a navigation landmark.
  • Table headers — Converts the first row of data tables to proper <th> header cells.
  • Table captions — Adds descriptive captions to tables that have none.
  • Fake lists — Replaces lines that use manual bullet characters or numbers with proper <ul> / <ol> list markup.
  • Color contrast — Adjusts foreground text colors that fail the WCAG AA contrast ratio against their background.
  • Image alt text AI — Generates descriptive alt text for images using an AI model (requires Anthropic key).
  • Link text AI — Rewrites non-descriptive links such as "click here" to meaningful text (requires Anthropic key).
PDF Files
  • Document metadata — Sets the document title and language, and marks the PDF as tagged for assistive technology.
  • Scanned PDF (OCR) — Runs optical character recognition on image-only PDFs to add a searchable, readable text layer. Capped at 10 files per run.
  • Tag structure — Adds a minimal accessibility tag tree to untagged PDFs so screen readers can parse the content.
  • Heading levels — Ensures headings start at H1 and do not skip levels (e.g. H1 → H3 becomes H1 → H2).
  • Table headers — Promotes first-row table cells from TD to TH in the PDF structure tree.
  • Image alt text — Adds a placeholder alt-text description to figure elements that have none.
  • Link descriptions — Sets an accessible label on URL link annotations that currently have no description.
Word Documents (.docx)
  • Heading presence & order — Adds headings if absent and corrects level skips, mirroring the HTML page fixes.
  • Table headers — Marks the first row of each table as a header row using Word's built-in table style.
  • Document language — Sets the document language attribute so screen readers use the correct pronunciation rules.
  • Image alt text — Generates AI alt text (with Anthropic key) or inserts a human-review placeholder if no key is provided.
PowerPoint Files (.pptx)
  • Reading order — Reorders slide elements so tab order matches visual layout, preventing confusion for keyboard and screen reader users.
  • Slide titles — Adds a title placeholder to slides that have none, which assistive technology uses to announce each slide.
  • Presentation language — Sets the language on the presentation so speech synthesis uses the correct voice and rules.
  • Image alt text — Adds a placeholder description to images that have no alt text.
  • Link descriptions — Flags non-descriptive hyperlinks and replaces them with the destination URL as a fallback label.
Images (via Ally)
  • Decorative images — Marks purely decorative images (logos, dividers, etc.) as decorative in Ally so screen readers skip them.
  • Seizure-risk images — Detects and replaces flashing or high-contrast strobing images that could trigger photosensitive seizures.
